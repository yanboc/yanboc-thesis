\chapter{快速开始}
\label{chap1}

本章给出模板的快速启动流程，分为本地部署与线上部署两种方式。推荐优先使用本地部署（以 \glsfirst{cursor} 或 \glsfirst{vscode} 为例），因为可以获得更稳定的编译速度、清晰的日志定位和更完整的工程管理体验。

\section{本地部署}

本节以 VS Code 为例（Cursor 等同理），介绍在自己的电脑上部署本模板的流程。

\subsection{基础环境准备}

建议使用 \glsfirst{texlive}（或 \glsfirst{miktex}）并确保 \texttt{xelatex} 可用。
本模板默认中文环境与字体配置基于 \glsfirst{xelatex} 工作流，一次完整的编译链路如下：
\begin{mynotice}
\begin{enumerate}
  \item \texttt{xelatex}（首轮生成辅助文件）；
  \item \texttt{bibtex}（如有文献引用）；
  \item \texttt{glossaries}（如有术语表）；
  \item \texttt{xelatex}（回填文献与交叉引用）；
  \item \texttt{xelatex}（稳定目录、页码与引用）。
\end{enumerate}
\end{mynotice}
在实际使用中，建议使用下述编译链路以提高编译速度：
\begin{mynotice}
  \begin{enumerate}
    \item \texttt{xelatex}（首轮生成辅助文件）；
    \item \texttt{bibtex}（在文献引用发生变化时重新生成）；
    \item \texttt{glossaries}（在术语表发生变化时重新生成）。
  \end{enumerate}
  \end{mynotice}
在 VS Code 中，推荐安装 \gls{latexworkshop} 扩展并启用自动构建。
下述配置仅供参考：
\begin{itemize}
  \item 自动构建：\texttt{latex-workshop.latex.autoBuild.run = onSave}，保存后会自动触发最近一次配方；
  \item 默认配方：\texttt{latex-workshop.latex.recipe.default = lastUsed}，建议首次手动选择一次完整配方后固定使用；
  \item 查看方式：\texttt{latex-workshop.view.pdf.viewer = tab}，PDF 直接在编辑器标签页中打开。
\end{itemize}

推荐的构建配方可按场景使用：
\begin{mynotice}
\begin{enumerate}
  \item \texttt{xelatex-draft}（\texttt{xelatex + bibtex}）；
  \item \texttt{xelatex}（\texttt{xelatex + bibtex + gls + 2 * xelatex }）；
  \item \texttt{xelatex-bibtex-gls (draft)}（\texttt{xelatex + bibtex + gls}）。
\end{enumerate}
\end{mynotice}

与本模板最匹配的建议是：日常写作用 \texttt{xelatex-draft}，提交或发布前切换到 \texttt{xelatex} 全量构建；当术语表有更新时，使用包含 \texttt{makeglossaries} 的配方。

\subsection{编辑器工作流建议}

在 \gls{vscode} 中，推荐安装 \glsfirst{latexworkshop} 扩展并启用自动构建。典型流程如下：
\begin{enumerate}
  \item 打开项目根目录并定位主文件 \texttt{thesis.tex}。一般来说，\gls{latexworkshop} 会自动识别该文件并将其设为根文件（大概是通过查找 \texttt{begin\{document\}} 命令）。
  \item 触发构建并观察问题 \texttt{PROBLEMS} 与输出 \texttt{OUTPUT} 面板（\texttt{CTRL+J}唤出）；
  \item 若编译失败，建议先检查路径（文件、图片等）、字体与引用标签是否有效。
\end{enumerate}

\subsection{配置文件解析}

模板的配置入口集中在 \texttt{settings/\_\_settings\_\_.tex}，其余文件按功能拆分：
\begin{itemize}
  \item \texttt{settings/bib\_style.tex}：参考文献样式与条目间距；
  \item \texttt{settings/cleveref-chinese.tex}：中文交叉引用名称与格式；
  \item \texttt{settings/design.tex}：中文字体、颜色与提示框样式；
  \item \texttt{settings/gls.tex}：术语索引与缩略语系统；
  \item \texttt{settings/plotting.tex}：\glsfirst{tikz}/\glsfirst{pgfplots} 绘图配置；
  \item \texttt{settings/shortcuts.tex}：常用宏、算子与算法环境；
  \item \texttt{settings/quick\_compile.tex}：快速编译开关与占位图策略。
\end{itemize}

建议优先在上述文件中完成个性化配置，而不是在正文中临时覆盖，以保持模板可维护性。

\section{线上部署}

\gls{overleaf} 的优势是免安装、可共享与快速演示。
用户可以通过将本模板的压缩包上传至 \gls{overleaf} 并解压来使用。
然而，在大型中文论文场景中，在线编译可能受到网络与资源限制，容易出现构建耗时长甚至超时（这对于免费用户而言几乎是必然的）。
此外，\gls{overleaf} 不支持绝大多数 AI 功能（特别是 agentic 的那些）。
因此，本模板不建议将 \gls{overleaf} 作为主力写作环境。

\begin{mynote}{线上部署的适用场景}
\begin{itemize}
  \item 跨设备临时查看（这个功能可以通过 GitHub 私有仓库实现）。
  PS. 我个人非常推荐项目化管理自己的每一篇论文；
  \item 与合作者共同编辑、修改与审阅（也许这更常见于小论文场景，因为学位论文主要由自己完成，很少有导师会直接帮忙改源文件）；
  \item TikZ 画图：我觉得这是最值得推荐的功能。在 \gls{overleaf} 中，你可以更专注于当前的图片，而且不用担心引入太多宏包而拖慢编译速度。
  将在 \gls{overleaf} 中使用 \texttt{standalone} 模式画好的 PDF插入大论文几乎不会影响编译速度。
\end{itemize}
\end{mynote}

\paragraph{风险与建议}
当章节、图片和引用规模增大时，在线编译的稳定性通常弱于本地编译；
同时，在线编译的可定制化水平也相对较低。
在线环境字体与工具链版本可能与本地不一致；
若必须在线协作，建议保留本地作为主分支并定期同步。
