本章说明本模板的主要参考来源与核心特色，便于读者快速理解模板设计思路，并据此进行二次定制。

\section{本模板的参考}\label{sec:reference}

本模板的结构、文档组织方式和武汉大学论文场景适配，主要参考以下公开项目与页面：
\begin{itemize}
  \item WHUTUG 模板：\url{https://github.com/whutug/whu-thesis}；
  \item Overleaf 模板（源自黄正华老师）：\url{https://www.overleaf.com/latex/templates/wu-yi-da-xue-bo-shi-lun-wen-latex-mo-ban/rcdzgvqgkddk}。
\end{itemize}

上述参考为本模板提供了结构化组织与基础接口设计启发；本工程在此基础上进一步强调“配置集中、模块拆分、可维护扩展”的工程化实践。

\section{特色功能（\texttt{settings/} 简介）}

\texttt{settings/} 目录承担模板的核心配置职责，目标是将“内容写作”与“排版控制”解耦。其主要特性如下。

\subsection{集中入口与模块拆分}

\begin{itemize}
  \item \texttt{\_\_settings\_\_.tex} 作为统一入口，顺序加载各子配置；
  \item 每个子文件职责单一，便于定位问题与局部替换；
  \item 推荐通过该入口集中管理，不在正文分散 \texttt{\textbackslash usepackage}。
\end{itemize}

\subsection{参考文献与交叉引用中文化}

\begin{itemize}
  \item \texttt{bib\_style.tex} 支持国标文献样式与条目间距调节，可用于管理文献引用。
  作为例子，这里我们引一下 Transformer\cite{DBLP:conf/nips/VaswaniSPUJGKP17} 与 GPT-3\cite{DBLP:conf/nips/BrownMRSKDNSSAA20}）；
  \item \texttt{cleveref-chinese.tex} 基于 \glsfirst{cleverefpkg} 统一图、表、公式、章、节等中文引用输出；
\end{itemize}

\begin{remark}[已知问题]\label{rem:known-problem}
  本模板中所使用的 \texttt{cleveref} 和 \texttt{gls} 包均与 \texttt{ctex} 包存在一定的兼容性问题。
  其中，最容易发现的问题就是交叉引用之前与之后的空格。
  这里给出一个例子：比如使用gls引用\gls{cleverefpkg}和直接写cleveref的名字在排版上的区别是很明显的。
  另一个例子是\cref{rem:known-problem}，当\texttt{cref}命令后直接接标点符号时，会出现空格错误。
  正确的格式当为注 \ref{rem:known-problem}。
  本模板暂时没有解决这个问题，所以建议用户在使用\texttt{gls}和\texttt{cleveref}时要格外留意其中的英文和数字。
\end{remark}

\subsection{版式、术语与绘图协同}

\begin{itemize}
  \item \texttt{design.tex} 管理中文字体、颜色和盒子（主要是\texttt{mynote}和\texttt{mynotice}）。
  字体方面，模板默认使用\href{https://ctan.org/tex-archive/fonts/fandol?lang=en}{Fandol 字体族}，常用切换命令包括 \texttt{\textbackslash songti}（宋体）、\texttt{\textbackslash heiti}（{\heiti 黑体}）、\texttt{\textbackslash kaishu}（{\kaishu 楷书}，即斜体）。
  例如：\texttt{{\textbackslash heiti 这是黑体标题示例}}。
  颜色方面，已预设 \texttt{deepblue-background}、\texttt{darkgray-background} 等颜色变量，可用于统一盒子或图形配色，避免正文中零散写 RGB 参数；
  \item \texttt{plotting.tex} 预设 \gls{tikz} 与 \gls{pgfplots} 库及常用样式；
  \item 可在不改正文的前提下统一调整视觉风格。
\end{itemize}

\subsection{术语系统与写作效率}

\begin{itemize}
  \item \texttt{gls.tex} 基于 \glsfirst{glossariespkg} 宏包，负责术语管理与索引。
  术语条目于 \texttt{includefile} 文件夹中的 \texttt{gls\_cmd.tex} 中定义。
  正文中第一次出现建议用 \texttt{\textbackslash glsfirst\{key\}}，后续用 \texttt{\textbackslash gls\{key\}}，即可自动统一术语显示与索引；
  \item \texttt{shortcuts.tex} 提供数学符号的快捷键（可选，但推荐使用）。
  例如可直接使用 \texttt{\textbackslash relu := \textbackslash rm{ReLU}}、\texttt{\textbackslash R := \textbackslash mathbb\{R\}}、\texttt{\textbackslash x := \textbackslash boldsymbol\{x\}} 这类高频缩写，减少重复输入（提高效率）并保持全文符号一致；
  \item \texttt{quick\_compile.tex} 提供快速编译开关，核心原理是“减少重资源输入”：
  在快速模式下，将大图与部分绘图输入替换为占位资源（\texttt{figures/dummy.pdf}），从而减少 I/O 与绘图渲染时间。
  这适合高频改稿阶段；定稿前切回正常模式即可恢复真实图片与完整版式。
\end{itemize}

综上，\texttt{settings/} 的核心价值在于：通过模块化配置实现统一风格、稳定编译和可持续维护。
