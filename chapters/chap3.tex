\chapter{使用说明}\label{chap3}

本章给出模板的常见自定义方法，覆盖字体、章节样式、定理环境与参考文献整理流程，便于在保持模板稳定性的前提下完成个性化写作。

\section{调整字体}

字体配置集中在 \texttt{settings/design.tex}。推荐原则是“先改映射，再改调用”：
\begin{itemize}
  \item 优先调整 \texttt{\textbackslash setCJKmainfont}、\texttt{\textbackslash setCJKsansfont}、\texttt{\textbackslash setCJKmonofont}；
  \item 确保 \texttt{fonts/} 中的字体文件名与配置一致；
  \item 修改后优先检查封面、章节标题、正文和图注是否统一。
\end{itemize}

若仅需局部切换字体，可使用模板已有命令（如 \texttt{\textbackslash songti}、\texttt{\textbackslash heiti}、\texttt{\textbackslash fangsong}、\texttt{\textbackslash kaishu}），避免在正文中重复定义新命令。

\section{自定义章节与定理等环境格式}

\subsection{章节层级与标题样式}

章节样式主要在 \texttt{settings/\_\_settings\_\_.tex} 中通过 \texttt{\textbackslash ctexset} 控制，可调整内容包括：
\begin{itemize}
  \item 章、节、小节的字号与加粗方式；
  \item 章节编号样式（如中文“第X章”）；
  \item 标题对齐方式与间距。
\end{itemize}

建议仅修改必要字段，并在每次改动后全量编译一次，确认目录与交叉引用未受影响。

\subsection{定理类环境}

定理、引理、命题、定义、例、注等环境的显示格式与引用名称通常通过两处联动控制：
\begin{enumerate}
  \item 文档类与正文中的环境定义（环境本体）；
  \item \texttt{settings/cleveref-chinese.tex}（引用名称与输出格式）。
\end{enumerate}

如果新增环境（如 \texttt{assumption}），建议同步完成：
\begin{itemize}
  \item 增加环境定义；
  \item 在 \texttt{cleveref-chinese.tex} 中补充 \texttt{\textbackslash crefname} 与 \texttt{\textbackslash Crefname}；
  \item 视需要补充 \texttt{\textbackslash crefformat} 多引用与区间引用规则。
\end{itemize}

\section{自动整理参考文献}

本模板提供“脚本去重 + 编辑器工具链”两类文献维护方式，推荐组合使用。

\subsection{脚本去重（\texttt{refs/prepare\_ref.py}）}

\texttt{refs/prepare\_ref.py} 可对 \texttt{refs/ref.bib} 执行基于条目 ID 的去重。典型流程如下：
\begin{enumerate}
  \item 先备份 \texttt{ref.bib}；
  \item 运行脚本清理重复项；
  \item 手动复核关键条目（作者、年份、期刊/会议字段）。
\end{enumerate}

该脚本适合日常维护，尤其在多来源合并 Bib 文件时可快速降低重复率。

\subsection{LaTeX Workshop 内置能力}

在 \gls{vscode} 中，可结合\gls{latexworkshop}的构建与日志能力管理文献：
\begin{itemize}
  \item 在保存时自动编译；
  \item 通过日志快速定位缺失引用、未定义条目和样式问题；
  \item 与脚本流程配合，形成“先去重、后编译、再校验”的闭环。
\end{itemize}

实践上，建议每次批量新增文献后先运行一次 \texttt{prepare\_ref.py}，再执行完整编译，确保引用编号与文献列表稳定。

\subsection{AI 工具}

本模板在制作过程中大量依靠 AI 工具，包括：
\begin{itemize}
  \item \textbf{Agent 偏好与任务记录}：
  在 \texttt{agent/} 目录中集中记录协作偏好、主要任务与阶段性待办（如偏好文件、模板化任务清单、协作说明），使 AI 在连续多轮修改中保持一致输出风格与执行边界；
  \item \textbf{目录级职责声明}：
  在每个主要目录放置 \texttt{\_agent.md}（如 \texttt{refs/\_agent.md}），向 AI 说明该目录的功能定位、可修改范围与维护规则，降低跨目录误改风险；
  \item \textbf{Agentic 项目管理思路}：
  采用“先定义规则、再批量改造、最后回归校验”的流程推进模板重构。
  具体地，先在 \texttt{agent/} 明确目标与约束，再按目录逐步替换内容并补充文档，最后通过统一编译与检查日志完成验收；
  \item \textbf{可追踪的迭代方式}：
  将任务拆分为可独立验证的小步骤（例如先处理摘要、再处理章节、再处理参考文献），每一步都保留可回溯的文本记录，便于后续继续协作或交接维护。
\end{itemize}
