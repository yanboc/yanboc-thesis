更多相关工作请读者参考 \ref{chap5-subsec:related work unlearning-based alignment}。

与这些方法相比，一些最近的工作在训练后阶段微调模型以遗忘争议信息，这提供了更彻底的防御 \cite{DBLP:journals/corr/abs-2404-05868,DBLP:conf/nips/YaoXL24,DBLP:conf/uss/SongKK0S25}。

本文致力于研究基于遗忘的对齐方法中的语义移除机制。
通过探测对齐后模型的层后逻辑值（post-layer logits），我们发现了一个有趣的现象，称为\textit{隐式唤起}（implicit evocation），即模型似乎已经遗忘了争议信息，但在中间层仍然隐式地唤起相关记忆。
例如，假设要遗忘的目标信息是{``斯蒂芬·金出生于波特兰''}（Stephen King was born in Portland）。
对齐后，当用{``斯蒂芬·金出生在缅因州的哪个城市''}（In which city in Maine was Stephen King born）提示模型时，尽管模型最终没有输出"波特兰"（Portland）作为答案，但"波特兰"的预测概率在中间层仍然显著上升。
如 \ref{chap5-fig:linear probing results} 所示，在不同基线上的实验表明，隐式唤起是基于遗忘的对齐方法中的普遍现象，这表明现有方法尚未实现彻底的遗忘。

作为我们的第一个关键贡献，我们从理论上解释了隐式唤起的根本原理。
受 \citet{DBLP:conf/naacl/MerulloEP24} 的启发，我们从残差流的角度分析 LLM 的遗忘过程。
我们采用已建立的\textit{线性表示假设}（Linear Representation Hypothesis, LRH）\cite{DBLP:conf/blackboxnlp/NandaLW23,DBLP:conf/nips/WangGNV23,DBLP:conf/icml/ParkCV24,DBLP:conf/iclr/0001CJV25}，并将 LLM 中的语义信息表征为\textit{词向量和概念向量}。
% detailed related works are postponed to \ref{chap5-{subsec:related work theory}.
如 \ref{chap5-fig:residual stream} 所示，Transformer 块中的残差连接通过向隐藏状态添加抽象的``概念向量''来细化隐藏状态。
粗略地说，我们的理论分析表明，来自语义无关标记的信号（以正交性为特征）不会显著影响与目标词相关的那些中间概念。
然而，这些信号仍然会扰动其他概念，从而改变模型的输出以实现显式遗忘，同时保留关于目标词的信息。

我们的理论分析得出以下两个\textit{指导原则}（参见 \ref{chap5-thm:1} 和 \ref{chap5-thm:2}）：
\begin{enumerate}
    \item 在遗忘过程中引入语义无关的标记会增加隐式唤起现象的可能性，以及
    \item 简单地对训练数据应用均值池化可以有效缓解隐式唤起。
\end{enumerate}

% that circumvents the complex procedures of explicit reward modeling and online reinforcement learning; 
% see \ref{chap5-{subsec:related work PO} for further details on related work.
主要遵循上述原则，我们提出了一种缓解隐式唤起的新策略。
我们采用类似于 \citet{DBLP:conf/nips/RafailovSMMEF23} 的\textit{负偏好优化}（negative preference optimization, negative PO 或 NPO）框架，并在遗忘过程中融入全局语义信息。
这里，PO 是一种基于偏好信号的对齐策略；相关工作的详细信息请参见 \ref{chap5-subsec:related work PO}。
我们提出的方法名为\textit{语义负偏好优化}（Semantic Negative Preference Optimization, SeNPO），从中间层提取句子级嵌入，并与预训练的嵌入模型进行比较以推导偏好信号。
我们在各种基准测试上进行了大量实验，以验证所提出方法的有效性。 

\paragraph{贡献声明} 
总之，本文的贡献如下。
\begin{itemize}
    \item 我们在各种基于遗忘的对齐方法中发现了一个普遍现象，称为隐式唤起，识别出现有方法的潜在局限性（参见 \ref{chap5-subsec:explore implicit evocation} 和 \ref{chap5-subsec:experiments details linear probing}）。
    \item 此外，本文从理论上阐明了隐式唤起现象背后的原理。 
    我们不仅提出了直观的见解和严格的理论证明，还提炼出两个指导原则，为未来的方法设计提供信息（参见 \ref{chap5-subsec:explain implicit evocation} 和 \ref{chap5-sec:theory}）。
    \item 基于理论启发的原则，我们提出了一种新的对齐策略，称为 SeNPO，它有效缓解了隐式唤起，并在标准基准测试上达到了接近最先进的性能，为构建更安全的 LLM 提供了新见解（参见 \ref{chap5-subsec:SeNPO} 和 \ref{chap5-sec:experiments}）。
\end{itemize}

\section{相关工作}\label{chap5-sec:related works}

本节讨论基于遗忘的对齐和 PO 框架的相关工作。

\subsection{基于遗忘的对齐}\label{chap5-subsec:related work unlearning-based alignment}
% @wsh(done): (i.e., jailbreak and harmful fine-tuning) 这里, jailbreak 后面 cite AutoDAN, PAIR, GCG, Harmful FT 后面 cite Vaccine 和 T-Vaccine
尽管对齐技术快速发展，针对 LLM 的两种主要对抗攻击形式（即越狱攻击 \cite{DBLP:journals/corr/abs-2307-15043,DBLP:conf/satml/ChaoRDHP025,DBLP:conf/iclr/LiuXCX24} 和有害微调 \cite{DBLP:conf/nips/HuangH024,DBLP:journals/tifs/LiuLMHMTS25}）仍然在实际应用中给 LLM 带来重大安全风险。
在本文中，我们专注于针对越狱攻击的安全对齐。
% , in which the adversaries leverage maliciously-crafted jailbreak prompts to bypass safety guardrails of LLMs \cite{DBLP:journals/corr/abs-2307-15043,DBLP:conf/ccs/ShenC0SZ24} and force the aligned LLMs to generate objectionable (e.g., harmful, biased, or misleading) content.
% To mitigate these threats, extensive research has focused on LLM defense, alignment, and guardrails in recent years. 
现有解决方案主要分为两类。 
% @wsh(done): rule-based content filtering, prompt engineering constraints 分别给cite，如果找不到可以替换或删掉
第一类方法专注于输出或逻辑值级别的干预 \cite{DBLP:journals/corr/abs-2212-08073}，包括基于规则的内容过滤 \cite{DBLP:conf/ccs/WuDPCXL024}、提示工程约束 \cite{DBLP:journals/corr/abs-2404-13208} 和响应验证 \cite{DBLP:conf/nips/MadaanTGHGW0DPY23}。 
这些方法高效（因为不需要进一步训练）并在许多基准测试中有效。 
然而，对齐后的 LLM 仍然容易受到未见过的攻击 \cite{DBLP:conf/icml/WeiHHXQXMW024,DBLP:conf/coling/MeiLWBMC25,DBLP:conf/icml/ZhaoCSHLMS25}，因为争议知识仍然存在于其内部表示中。

第二类，也是更根本的类别，是基于遗忘的对齐，旨在从 LLM 参数中消除不良知识，从源头解决安全风险，而不仅仅是抑制有问题的输出。
% Unlearning-based alignment emerges as a promising paradigm for LLM safety, with diverse techniques in the literature. 
早期方法主要依赖于权重编辑 \cite{DBLP:conf/acl/MelaGHV24,DBLP:journals/corr/abs-2505-20819,DBLP:journals/corr/abs-2505-19855}，直接修改预训练 LLM 的特定权重矩阵以根除目标有害知识。 
它们在单事实遗忘方面在效率和精度上表现出色，但缺乏大规模/抽象有害知识的可扩展性。 
另一个分支使用以数据集为中心的策略 \cite{DBLP:journals/corr/abs-2310-02238}，即在带有有害反例的自定义遗忘数据集上微调 LLM \cite{DBLP:conf/icml/LiPGYBGLDGMHLJL24}，引导模型用安全知识替换不良知识。
然而，这需要大量标注，并存在灾难性遗忘良性知识的风险。 
最近的进展将 PO 整合到基于遗忘的对齐中，产生了如 NPO \cite{DBLP:journals/corr/abs-2404-05868} 等方法。 
这些框架将遗忘视为偏好学习任务，使用负偏好信号引导模型丢弃有害知识，同时保持一般性能。 
尽管取得了显著进展，但大多数现有方法在标记级别操作，无法捕获句子级语义，在处理改写的有害内容和过度遗忘方面存在困难。

\subsection{偏好优化}\label{chap5-subsec:related work PO}
PO 构成了将预训练 LLM $\pi_\theta$ 微调到人类偏好的核心对齐范式 
\cite{DBLP:conf/eurosp/ThudiDCP22,DBLP:conf/iclr/PatilHB24,DBLP:conf/nips/RafailovSMMEF23,DBLP:journals/corr/abs-2404-05868}，为 NPO \cite{DBLP:conf/nips/RafailovSMMEF23} 奠定了理论基础。
它依赖于一个配对偏好反馈数据集，其中偏好关系通过人工评估得出。
PO 的主要目标是利用配对偏好反馈数据集细化模型参数，从而增强其与人类偏好标准的对齐。

PO 作为推进基于遗忘的对齐的关键基石，其内在优点解决了先前方法的瓶颈。

数据效率、训练稳定性和语义级对齐的协同作用巩固了 PO 在 LLM 安全对齐中的核心地位。

\subsection{预备知识}\label{chap5-subsec:preliminaries}
% We start with introducing the basic setups.
% Throughout this paper, we use a parenthesized superscript $\x^{(i)}$ to denote the $i$-th entrance of an array-like variable $\x$ (e.g., vectors and matrices).
% Throughout this paper, we use boldface lowercase (e.g., $\x$) and uppercase (e.g., $\W$) letters to denote vectors and matrices, respectively.
% Given $d \in \mathbb{N}^{+}$ and $\x \in \R^d$, the $i$-th ($1 \leq i \leq d$) entry of $\x$ is denoted by $\x^{(i)}$, i.e., $\x = (\x^{(1)}, \x^{(2)}, \cdots, \x^{(d)})^{\top}$.
% Similarly, we use $\W^{(i)}$ to denote the $i$-th column of a matrix $\W$ and write $\W = (\W^{(1)}, \W^{(2)}, \cdots, \W^{(d)})$.
% Given $N \in \mathbb{N}^{+}$, the index set $\{1, 2, \cdots, N\}$ is noted by $[N]$.
% For any $n \in [N]$, we let $[N \backslash n] := \{ 1, \cdots, n-1, n+1, \cdots, N \}$ for simplicity.

% We start with the sconcepts of LLMs.
% $\V = (v_1, v_2, \cdots, v_{|\V|})$
设 $\V$ 为所有可能标记的集合（即\textit{词汇表}集合）。
% $\x = (\x^{(1)}, \x^{(2)}, \cdots, \x^{(L)}) \in \V^{L}$
给定长度为 $L \in \mathbb{N}^{+}$ 的标记序列 $\x \in \V^{L}$，因果 LLM 的目标是预测 $\x$ 的下一个标记（或\textit{标签}），记为 $y \in \V$。 
% @wsh(done): Qwen cite Qwen3 technical report, Dpsk cite V3 tech report
典型的因果 LLM（例如，Qwen \cite{DBLP:journals/corr/abs-2505-09388} 和 DeepSeek \cite{DBLP:journals/corr/abs-2412-19437}）可以分解为\textit{特征提取器} $\h_{\theta}$ 和\textit{语言模型头}（或\textit{预测头}）$\lambda$，
% $\lambda: \mathbb{R}^{d_h} \to \mathbb{R}^{|\V|}$。
其中，$\theta$ 表示 $\h$ 的可训练参数。
LM 头 $\lambda$ 的参数不是我们分析的核心，因此未指定。
% Let $d_h$ be the dimension of the hidden state.
设 $M$ 为 LLM 的深度。
那么，特征提取器可以进一步分解为 $M$ 个 Transformer 块的堆叠，记为
\begin{equation}\label{chap5-eq:def transformer blocks}
    \h_\theta(\x) := \h_{\theta}^{(M)} \circ \h_{\theta}^{(M-1)} \circ \cdots \circ \h_{\theta}^{(2)} \circ \h_{\theta}^{(1)}(\x).
\end{equation}
这里，带括号的上标（例如，$\h^{(i)}$）用于索引类数组变量的第 $i$ 个元素。
模型的输出分布（或\textit{逻辑值}）由下式给出
\begin{equation}\label{chap5-eq:def logits dist}
    p_{\theta}(y = v_i \mid \x) = \frac{\left(\lambda \circ \h_{\theta} (\x)\right)^{(i)}}{\sum_{j \in [|\V|]} \left(\lambda \circ \h_{\theta} (\x)\right)^{(j)} },
\end{equation}
输出标记 $v$ 由下式给出
\begin{equation}
     v = \argmax_{v_i \in \V} p(y = v_i \mid \x).
\end{equation}

\subsection{探索隐式唤起现象}\label{chap5-subsec:explore implicit evocation}
设 $\theta_0$ 为预训练 LLM 的参数。 
基于遗忘的对齐的主要目标是迫使 LLM 遗忘争议信息，同时保留其他信息。
更具体地说，设 $\D_{FG}$ 和 $\D_{RT}$ 分别为遗忘和保留数据集。
设 $R(\theta, \mathcal{D})$ 为模型（由 $\theta$ 参数化）在数据集 $\D$ 上的风险。
直观地说，基于遗忘的对齐算法的共同目标是最大化 $R(\theta, \mathcal{D}_{FG})$，同时最小化 $R(\theta, \mathcal{D}_{RT})$。
值得注意的是，$\mathcal{D}_{RT}$ 通常非常大（考虑到 $\mathcal{D}_{RT} \cup \mathcal{D}_{FG}$ 基本上是预训练数据集），因此在实际遗忘任务中并未实际使用。

为简单起见，我们考虑 $\D_{FG}$ 中的数据形式为 $(\x, y)$。
给定 $(\x_0, y_0) \in \D_{FG}$，如果
\begin{equation}\label{chap5-eq:def successfully unlearn}
    y_0 \neq \argmax_{v \in \V} p_{\theta}(y = v \mid \x_0).
\end{equation}
我们说 $\h_{\theta}$ \textit{显式遗忘} $(\x_0, y_0)$。
注意，\ref{chap5-eq:def successfully unlearn} 作为端到端准则。
早期层的推理，即 $\h_{\theta}^{(i)}$ ($1 \leq i \leq M-1$)，未被考虑。
为了进一步探索模型的语义移除机制，我们可以在每个 Transformer 层后插入 LM 头并观察层后逻辑值。
更具体地说，对于所有 $1 \leq m \leq M$，设 
\begin{equation}\label{chap5-eq:def h m 1}
    \h^{(m,1)}_\theta(\x) := \h_{\theta}^{(m)} \circ \h_{\theta}^{(m-1)} \circ \cdots \circ \h_{\theta}^{(2)} \circ \h_{\theta}^{(1)}(\x)    
\end{equation}
为第 $m$ 层后的隐藏状态，且
\begin{equation}\label{chap5-eq:def post-layer logits}
    p_{\theta}^{(m,1)}(y = v_i \mid \x) = \frac{\left(\lambda \circ \h^{(m,1)}_{\theta} (\x)\right)^{(i)}}{\sum_{j \in [|\V|]} \left(\lambda \circ \h^{(m,1)}_{\theta} (\x)\right)^{(j)} },
\end{equation}
为\textit{第 $m$ 层后逻辑值}。
通过层后逻辑值检查隐藏状态中编码的上下文语义信息的方法在文献中通常称为线性探测 \cite{DBLP:conf/icml/Allen-ZhuL24}。

在本文中，我们通过标记 $v$ 在层后逻辑值中的\textit{倒数排名} $r^{(m,1)}_\theta$ 来测量 $v$ 的唤起，这指的是 $v$ 在逻辑值分布中的序数排名的倒数。
简单地说，如果 \(v\) 在 \(p^{(m,1)}_\theta(v)\) 中排名第 \(n\) 位，那么 \(v\) 的倒数排名为 \(r^{(m,1)}_\theta(v) = 1/n\)。
有了上述定义和度量，我们现在可以如下定义隐式唤起现象。

% As a warm-up, we first introduce the DPO and NPO strategies.
% Given hyperparameter $\beta$ (which represents the ``inverse temperature''), the objectives of DPO and NPO are given by \ref{chap5-{eq:def DPO,eq:def NPO}.
% Here, $\sigma(t) = 1/(1+e^{-t})$ is the sigmoid function.
% According to \citet{DBLP:conf/nips/RafailovSMMEF23}, each entrance of the DPO's dataset $\D_{paried}$, called a ``paired response dataset'', comprises a prompt $\x$ and a pair of positive and negative responses $y_+$ and $y_-$.
% Both responses are generated by the model with parameter $\theta_0$.
% The subscripts ``+'' and ``-'' are assigned by human annotators, implying a preference for $y_+$ over $y_-$.
% Consequently, the DPO objective defined in \ref{chap5-{eq:def DPO} aligns the model with human preferences.

\begin{definition}[隐式唤起]
\label{chap5-def:implicit evocation}
    如果
    \begin{equation}
        r_{\theta}^{(M,1)}(y) < 1,
    \end{equation}
    即目标输入 $y$ 未被显式输出，且对于某个阈值 $\tau \in (0,1)$，
    \begin{equation}
        r_{\theta}^{(m,1)}(y) > \tau
    \end{equation}
    这意味着 $y$ 的预测概率出现意外增加，则我们说模型 $\h_{\theta}$ 在级别 $\tau$ 下隐式唤起 $(\x, y) \in \mathcal{D}_{FG}$。\hfill $\blacktriangle$
\end{definition}
\ref{chap5-def:implicit evocation} 表明，如果对齐后的 LLM 表现出隐式唤起现象，模型仍可能（或被强制）生成争议内容。
一种直接的红队测试方法是提取层后逻辑值，并采用高温度下的 top-$(1/\tau)$ 采样。

在本文中，我们检查了通过各种基于遗忘的对齐方法对齐的模型的层后逻辑值。
如 \ref{chap5-fig:linear probing results} 所示，我们发现隐式唤起现象在所考虑的方法中普遍存在，这表明现有方法尚未实现彻底遗忘。

\subsection{理论分析：概述} \label{chap5-subsec:explain implicit evocation}
% \input{figures/DPO_NPO_SeNPO}

本节提供了隐式唤起现象的直观解释，并提出了有助于缓解该现象的两个原则。

\paragraph{分析框架} 理解该现象的关键在于建模和分析语义信息流。
采用已建立的线性表示假设（Linear Representation Hypothesis, LRH）\cite{DBLP:conf/blackboxnlp/NandaLW23,DBLP:journals/corr/abs-2308-10248,DBLP:conf/nips/WangGNV23}，我们将标记表征为\textit{``词向量''}。
稍微滥用记号，我们设 $\boldsymbol{V} := (\v_1, \v_2, \cdots, \v_{|\V|})^{\top}$ 为词汇表集合的嵌入矩阵，使得对于所有 $i \in [|\V|]$，$\v_i \in \mathbb{R}^{d_h}$ 是 $v_i$ 的嵌入向量。
% As demonstrated in \ref{chap5-{fig:SeNPO}, we model the unlearning task from the perspective of the residual stream and analyze the dynamics of the iterations in the unlearning process.
对于模型，我们将第 $m$ 个 Transformer 块（参见 \ref{chap5-eq:def transformer blocks}）指定为
\begin{equation}\label{chap5-eq:def transformer blocks specified}
    \h^{(m)}_{\theta}(\u) = \u + \W^{(m)}_{\theta} \attn^{(m)}_{\theta}(\u)
\end{equation}
对于所有 $1 \leq m \leq M$。
结合 \ref{chap5-eq:def h m 1} 和 \ref{chap5-eq:def transformer blocks specified}，我们可以将相邻隐藏状态之间的差异写为
\begin{equation}\label{chap5-eq:def concept vector}
\begin{aligned}
    \c^{(m)}_{\theta}(\x)
    :=\ &\h^{(m,1)}_{\theta}(\x) - \h^{(m-1,1)}_{\theta}(\x) \\
    =\ & \W^{(m)}_{\theta} \attn^{(m)}_{\theta}(\h_{\theta}^{(m,1)}(\x))  
\end{aligned}  
\end{equation}
对于任何 $1 < m \leq M$。
值得注意的是，该向量聚合了来自早期层隐藏状态（即 $\h_{\theta}^{(m,1)}(\x)$）的语义，这些语义由当前层的注意力和 MLP 模块（即 $\W^{(m)}_{\theta} \attn^{(m)}_{\theta}$）处理。
我们将该向量称为该层的\textit{概念向量}。
\ref{chap5-fig:residual stream} 可视化了残差流以及词向量和概念向量。

上述分析反映了 Transformer 块通过向隐藏状态添加``概念向量''来细化隐藏状态。
这种机制与 Word2Vec \cite{DBLP:conf/naacl/MikolovYZ13} 有相似之处，但在表述和行为上存在根本区别。
我们在 \ref{chap5-app:word2vec} 中提供了我们的理论构造与 Word2Vec 的详细比较。

\paragraph{主要结果}
基于残差流的视角以及概念向量和词向量的概念，我们现在能够解释隐式唤起现象背后的原理。
由于 LLM 的自回归训练特性，遗忘过程不可避免地引入了大量与目标遗忘的核心语义无关的标记。
如果我们将这种无关性解释为目标词向量与中间概念向量之间的正交性，那么一个直观的结论就随之而来。 
由于正交性，这些无关标记不会影响与目标词相关的那些中间概念。 
然而，它们确实会扰动其他层的概念向量。 
特别是，如果最后几层的概念被扰动，它们很可能在中间概念向量发生显著变化之前改变模型输出。
我们将上述分析总结为以下声明。

\begin{proposition}[\ref{chap5-thm:1} 的非正式版本]\label{chap5-proposition:1}
     在遗忘过程中引入语义无关的标记会增加隐式唤起现象的可能性。
\end{proposition}

如上述分析所述，与核心语义无关的标记会增加隐式唤起的可能性。 
一个临时补救措施是仅在 $\D_{FG}$ 中包含语义相关的标记。
然而，语义信息通常在句子级别编码，无法用少数几个单独的标记来表示。
因此，作为替代方案，将句子级语义融入遗忘目标成为一种自然的方法。

句子级嵌入是一种成熟的技术，其核心原理在于对句子中的标记进行池化，例如最大池化、均值池化和首标记池化。
基于声明 \ref{proposition:1}，并考虑到遗忘任务的独特特性，均值池化似乎是唯一合适的方法。
一个潜在的担忧是，经过均值池化的稀释后，来自相关标记的信号（可能很少）能否有效缓解隐式唤起？
我们的定量分析提供了肯定的答案，总结为声明 2。

\begin{proposition}[\ref{chap5-thm:2} 的非正式版本]\label{chap5-proposition:2}
    简单地对训练数据应用均值池化可以有效缓解隐式唤起。
\end{proposition}

声明 \ref{proposition:1} 和 \ref{proposition:2} 为未来基于遗忘的方法缓解隐式唤起提供了两个指导原则。
更严格的分析可以在 \ref{chap5-sec:theory} 中找到，证明包含在 \ref{chap5-app:proof} 中。

\subsection{语义负偏好优化}\label{chap5-subsec:SeNPO}

我们通过提出一种遵循声明 \ref{proposition:1} 和 \ref{proposition:2} 的遗忘策略来结束本节，以缓解隐式唤起。

作为热身，我们首先介绍 DPO 和 NPO 策略。
给定超参数 $\beta$（表示``逆温度''），DPO 和 NPO 的目标由 \ref{chap5-eq:def DPO} 和 \ref{chap5-eq:def NPO} 给出。
这里，$\sigma(t) = 1/(1+e^{-t})$ 是 sigmoid 函数。
根据 \citet{DBLP:conf/nips/RafailovSMMEF23}，DPO 数据集 $\D_{paried}$ 的每个条目（称为``配对响应数据集''）包含一个提示 $\x$ 和一对正负响应 $y_+$ 和 $y_-$。
两个响应都由参数为 $\theta_0$ 的模型生成。
下标``+''和``-''由人工标注者分配，表示对 $y_+$ 的偏好优于 $y_-$。
因此，\ref{chap5-eq:def DPO} 中定义的 DPO 目标使模型与人类偏好对齐。

作为针对遗忘问题定制的 DPO 变体，NPO 将每个 $(\x, y) \in \D_{FG}$ 视为负响应。
通过从 DPO 的目标中省略正项，我们可以在 \ref{chap5-eq:def NPO} 中获得 NPO 的目标。

为了缓解隐式唤起现象，一种直观的方法是识别并移除问题和答案之间的高级语义连接。
受现有关于 LLM 可解释性工作的启发 \cite{DBLP:conf/icml/Bu0HN0WS25,DBLP:conf/icml/ParkCV24,DBLP:conf/iclr/0001CJV25}，我们将移除机制描述如下。

最终优化目标由下式给出
\begin{equation}\label{chap5-eq:def SeNPO with regularization}
    \alpha_1 R_{SeNPO}(\theta, \D_{FG})  + \alpha_2 K(\theta, \D_{FG})
\end{equation}
其中 $\alpha_1$ 和 $\alpha_2$ 是可调超参数。

\subsection{Experimental Setups}\label{subsec:exp setups implementation details}

\paragraph{Datasets}
We conduct experiments on the RWKU (Real-World Knowledge Unlearning) benchmark\footnote{https://huggingface.co/datasets/jinzhuoran/RWKU}, which is commonly used in unlearning-based alignments. 
The benchmark contains multiple target entities (e.g., Stephen King, Confucius, and Bruce Lee) for unlearning evaluation.
Each target entity includes positive samples for unlearning and evaluation datasets at three levels: forget level 1 (fill-in-the-blank tasks), forget level 2 (question answering), and forget level 3 (adversarial attacks). 
Additionally, the benchmark includes neighbor evaluation tasks to assess knowledge retention and utility evaluation tasks, including MMLU, BBH, TruthfulQA, TriviaQA, and fluency assessments.

\paragraph{Models}
    As mentioned in \ref{sec:methodology}, we consider three types of LLM.
    1). The \textit{pre-trained LLM} $(\lambda, \h_{\theta_0})$.
    We use Microsoft Phi-3-mini-4k-instruct\footnote{https://huggingface.co/microsoft/Phi-3-mini-4k-instruct} (3.8B parameters) as our base (unaligned) pre-trained model.
    The model has a hidden size of 3072 and uses the instruction-tuned variant for better alignment capabilities.
    2). The \textit{reference embedding model} $\boldsymbol{e}$. 
    We consider all-mpnet-base-v2 (4B, 768-dimensional) and Qwen3-Embedding-4B.\footnote{https://huggingface.co/sentence-transformers/all-mpnet-\\base-v2, https://huggingface.co/Qwen/Qwen3-Embedding-4B}
    3). The \textit{embedding head} $\eta$. We train a Transformer-based model with 6 layers, 8 attention heads, and a hidden dimension of 768 for the embedding model, which is consistent with the reference embedding model.
 
\paragraph{Baselines}
We compare our proposed method against the following three baselines.
1). \textit{Gradient Ascent (GA)}. 
GA performs unlearning by directly maximizing the loss on target samples, effectively performing gradient ascent on the unlearning objective. We use a learning rate of $6 \times 10^{-8}$ with AdamW optimizer and train for 2000 steps across 8 stages.
2). \textit{Negative Preference Optimization (NPO)} NPO applies negative preference optimization to reduce the model's preference for generating target knowledge. We set $\beta = 0.1$ and use a learning rate of $6 \times 10^{-8}$ with AdamW optimizer, training for 250 steps.
3). \textit{Align-then-Unlearn (ATU)} ATU performs unlearning in the semantic embedding space by first training an embedding head to anticipate future context representations, then fine-tuning the model to minimize the similarity between predicted embeddings and target embeddings representing the concept to be removed. We use a multi-stage alternating training procedure with progressively decreasing similarity thresholds (from 0.9 to 0.2). The embedding head is optimized using Adam optimizer with learning rate $10^{-4}$, while the LLM is optimized using SGD optimizer with learning rate $3 \times 10^{-4}$.

\paragraph{Implementation Details}
We implement SeNPO using a two-stage approach:
we first train the embedding head and then perform unlearning with a weighted sum of the SeNPO loss and a KL divergence regularization (cf. \ref{eq:def SeNPO with regularization}).
We set $\alpha_1 = 1.0$, $\alpha_2=0.1$, and $\beta = 1.0$ with training and unlearning learning rate of $10^{-4}$ and $3 \times 10^{-4}$, respectively.

We implement experiments using the PyTorch Lightning framework on NVIDIA A800 GPUs with 80GB of memory, consuming $10^{2}$ GPU-hours in total. 
We use Hydra for configuration management and can be launched either locally or via SLURM job scheduling system. 
We use mixed precision training (BF16-mixed) to optimize memory usage and training speed.
We refer the reader to \ref{app:exp details} for more details.

To characterize whether a model has already encoded the target answer (or answer-related cues) \textit{at different depths}, we adopt a lightweight \textit{linear probing} protocol. Concretely, we apply the model's own output linear map (\texttt{lm\_head}) to each layer's hidden state and measure how highly the target token is ranked in the resulting vocabulary logits.

% \begin{figure}[H]
%     \centering
%     \includegraphics[width=0.99\linewidth]{includefiles/Linear Probing.pdf}
%     \caption{A visualization of the linear probing process. We plug the LM head into the post-layer hidden states to observe the logits at the intermediate layers.}
%     \label{fig:linear probing}
% \end{figure}

\paragraph{Metric (Reciprocal Rank, RR).}
Given a prompt \(x\), let \(h^{(l)}\) denote the hidden state at layer \(l\) at the \textit{query position} (by default, the last non-special token of the prompt). We map \(h^{(l)}\) to vocabulary logits via the model head,
\(z^{(l)}=\texttt{lm\_head}(h^{(l)})\in\mathbb{R}^{|V|}\).
For a target keyword, we take its first token \(t\) (token id \(id(t)\)), compute its rank under the descending sort of \(z^{(l)}\), denoted \(rank^{(l)}(t)\), and define the layer-wise reciprocal rank
\[
RR^{(l)}(t) = \frac{1}{rank^{(l)}(t)} \in (0,1].
\]
Intuitively, a larger \(RR\) indicates a stronger ``awakening'' of the target token at that layer; \(RR=1\) means the target token is ranked top-1.

\paragraph{Implementation details.}
We run a forward pass on the prompt, capture per-layer hidden states, apply \texttt{lm\_head} at the query position to obtain logits, and compute \(1/rank\) by sorting over the full vocabulary. For multi-token keywords, we use the first token. We also optionally track control tokens such as \texttt{sorry} to contrast answer-related activation against refusal-related patterns.

We plot an answer-related token (e.g., \texttt{Portland}) together with a salient entity token (e.g., \texttt{stephen}) and a refusal-related token (e.g., \texttt{sorry}) to contrast how different layers prioritize answer cues versus refusal patterns.

\paragraph{Qualitative observations.}
Across all questions, we observe a broadly consistent depth profile. For the pretrained model, the RR of the answer token (\texttt{Portland}) remains near zero in early layers, rises sharply in the middle layers, and stays elevated towards the top, indicating that the correct answer becomes linearly decodable relatively early and remains so throughout the forward pass. After unlearning, the corresponding curves for \texttt{Portland} are substantially flattened, with middle and late-layer RR values noticeably reduced, while the RR of the entity token (\texttt{stephen}) is much less affected. At the same time, the refusal-related token (\texttt{sorry}) tends to have slightly higher RR in later layers for the unlearned models than for the pretrained one, suggesting that the intervention selectively suppresses linearly accessible knowledge about the forbidden fact while shifting probability mass toward a generic refusal mode rather than causing a uniform degradation of semantic representations.

\section{理论分析}\label{chap5-sec:theory}

本节为 \ref{chap5-sec:methodology} 中的声明提供理论保证---\ref{chap5-thm:1} 和 \ref{chap5-thm:2} 分别作为声明 \ref{proposition:1} 和 \ref{proposition:2} 的形式化版本。
为了完整性，我们重新陈述一些关键记号如下。

在我们的分析中，我们使用带括号的上标 $\x^{(i)}$ 来表示类数组变量 $\x$（例如，向量和矩阵）的第 $i$ 个入口。
给定 $N \in \mathbb{N}^{+}$，索引集 $\{1, 2, \cdots, N\}$ 记为 $[N]$。
对于任何 $n \in [N]$，为简单起见，我们设 $[N \backslash n] := \{ 1, \cdots, n-1, n+1, \cdots, N \}$。

为了便于表述和理论分析，我们考虑具有\textit{可分离概念}的 QA 任务。
这里，术语概念指的是影响 LLM 输出的转向向量，如果两个概念可以单独干预，我们说它们是可分离的。
接下来，我们使用 \ref{chap5-subsec:preliminaries} 中的记号来形式化这一直觉。
回想一下，隐藏状态的维度记为 $d_{h}$，这也表示大多数现代 LLM 中词标记嵌入的维度 \cite{DBLP:journals/corr/abs-2505-09388,DBLP:journals/corr/abs-2412-19437}。 
设 $\boldsymbol{V} := (\boldsymbol{V}^{(1)}, \boldsymbol{V}^{(2)}, \cdots, \boldsymbol{V}^{(|\V|)})$ 为词汇表集合的嵌入矩阵，使得对于所有 $i \in [|\V|]$，$\boldsymbol{V}^{(i)} \in \mathbb{R}^{d_h}$ 是 $v_i$ 的嵌入向量。

为简单起见，我们假设标记是良好分类的，即给定 $K \in \mathbb{N}^+$，对于所有 $v \in \V$，存在唯一的 $k \in [K]$，使得 $v$ 可以根据某个未指定的标准分类到第 $k$ 类。
用 $\V_k$ 表示第 $k$ 类中所有标记的集合。
然后我们可以写出概念的定义如下。

\begin{definition}[概念]
\label{chap5-def:concepts}
如果对于任何 $\v_1, \v_2 \in \V_k$，$\v_2 - \v_1$ 可以表示为 $\C$ 中向量的线性组合，则向量集合 $\C := \{\c_1, \c_2, \cdots, \c_n\} \subset \R^{d_h}$ 称为第 $k$ 类的\textit{类内概念}集合。 

类似地，如果对于所有 $\v_1 \in \V_{k_1}$ 和 $\v_2 \in \V_{k_2}$，$\v_2 - \v_1$ 可以表示为 $\C$ 中向量的线性组合，则我们说 $\C$ 是从第 $k_1$ 类到第 $k_2$ 类的\textit{类间概念}。

结合上述两个定义，如果 $\C$ 是类内概念或类间概念，我们将其表示为\textit{概念}集合。 \hfill $\blacktriangle$
\end{definition}

我们用 $\C_k$ 表示 $\V_k$ 的类内概念，用 $\C_{k_1, k_2}$ 表示从 $\V_{k_1}$ 到 $\V_{k_2}$ 的类间概念。
以下假设源于高维空间的稀疏性和概念的独立性\footnote{由于篇幅有限，我们将在 \ref{chap5-app:examples} 中通过一些示例性例子详细阐述这种独立性。}。

\begin{assumption}[参见 \citet{DBLP:conf/iclr/0001CJV25} 中的定理 8]
\label{chap5-assumption:orthogonal}
我们假设概念向量是成对正交的，即对于任何 $\c_1, \c_2 \in \boldsymbol{C}$，我们有 $\c_1 \perp \c_2$。 \hfill $\blacktriangle$
\end{assumption}
